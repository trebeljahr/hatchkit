Hatchkit handles scaffolding, providers, infrastructure, and deployment. These sibling tools cover the asset and naming work that often happens before or beside a Hatchkit project.

## Rico's dev-tools cluster

| Tool | What it does | Useful with Hatchkit when | Repo |
|---|---|---|---|
| `conv3d` | Converts FBX, OBJ, and glTF assets to GLB and can emit typed React Three Fiber components. | You scaffold a 3D app, game, or asset-heavy starter and need web-ready models before shipping. | [trebeljahr/conv3d](https://github.com/trebeljahr/conv3d) |
| `sprite-tools` | Builds game-ready 2D sprite assets: collision polygons, pivots, animation tags, palettes, atlases, and exports. | Your Hatchkit app has a game or interactive surface and needs repeatable sprite prep in CLI, web, or MCP form. | [trebeljahr/sprite-tools](https://github.com/trebeljahr/sprite-tools) |
| `name-checker` | Checks a candidate name across trademarks, domains, socials, app stores, package registries, and code hosts. | You are naming a new Hatchkit project before buying domains, creating repos, wiring DNS, or publishing packages. | [trebeljahr/name-checker](https://github.com/trebeljahr/name-checker) |
| `gamedev-asset-library` | Catalogs free, permissively licensed game assets across 3D models, sprites, sounds, music, and textures. | You want seed assets for a Hatchkit game, prototype, R3F scene, or ML-backed asset workflow. | [trebeljahr/gamedev](https://github.com/trebeljahr/gamedev) |
